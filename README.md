# CS-340
How I write programs that are maintainable, readable, and adaptable?
To make my code maintainable and adaptable, I used clear naming, modular functions, and kept logic separated in a reusable CRUD Python module. This structure made it easy to connect the backend to the dashboard in Project Two and will allow me to reuse or expand the module in future projects.
How do you approach a problem as a computer scientist?
I approached the dashboard requirements by breaking the problem into smaller tasks—designing the database, writing CRUD functions, and connecting them to visual components. Compared to earlier courses, this project felt more real-world, requiring integration of backend and frontend systems. In future projects, I’ll continue using modular and goal-driven development to meet client needs effectively.
What do computer scientists do, and why does it matter?
Computer scientists create tools that solve real problems. In this project, I helped Grazioso Salvare use their data more efficiently, improving decision-making and rescue operations. Projects like this show how programming can directly support important missions and make organizations more effective.
